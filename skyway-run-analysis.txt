skyway.run Analysis
====================

Author: Hank Brekke-Peer (@hank@ieji.de)
Source: Uses OpenStreetMap (OSM) data for the skyway paths
Technology: Custom JS app with MapLibre/Mapbox-style rendering

Key Data Source: OpenStreetMap
- The config contains OSM node IDs (e.g., 244884027, 245395408, etc.)
- The config contains OSM way IDs (e.g., startingWay: 413691183)
- Uses "roadMerges" with OSM way IDs
- Uses "simplifiedView" with OSM node IDs and coordinates

The app fetches actual OSM way/node data and renders the skyway paths.
The "startingWay" ID 413691183 is an OSM way that represents a skyway segment.

Main JS bundle: /assets/main-CX50CuVD.js
This bundle likely fetches OSM data via Overpass API or has it embedded.
